# プログラミング教室｜Canvaテンプレート仕様

糸かけアート教室の「表紙＋ステップ番号バッジ＋swipe誘導」という固定フォーマットの考え方を、プログラミング教室にも適用したもの。
カテゴリ（認知・価値提供・信頼構築・行動喚起）ごとに使うスライドパターンを整理し、毎回変えない**固定要素**と、投稿ごとに差し替える**可変要素**を分けて書き出す。

---

## 0. 全スライド共通の固定要素（デザインの土台）

| 要素 | 内容 |
|---|---|
| ロゴ位置 | 左上に固定配置（糸かけアートと左右非対称にして、教室を混同させない） |
| swipe誘導 | 右下に固定配置。矢印アイコン＋「SWIPE ▷▷」表記 |
| 配色 | ターコイズ×ビビッドオレンジの2色のみ（彩度高めのオレンジ、例：#FF6B35前後）。3色目を使う場合はどちらかの濃淡（トーン違い）に限定し、ネイビーや赤緑アクセントは使わない |
| NG/OK表現 | 色で判断させず、✕／○の記号のみで区別する |
| フォント | 見出し・英単語（Scratch, MakeCode, Roblox, Python等）は視認性の高い欧文フォントで強調。本文は和文ゴシック |
| キャンバス地 | 白〜オフホワイト基調。写真ベースの糸かけアートと違い、図解・SVG中心のクリーンな清潔感を出す |

## 1. スライドパターン一覧（カテゴリ対応）

| スライド種別 | 使うカテゴリ | 固定要素 | 可変要素 |
|---|---|---|---|
| 表紙／教室紹介カード | ①認知 | ロゴ位置、ベースカラー、見出しフォント、swipe誘導 | 見出しテキスト、キャッチコピー、アイキャッチ図形 |
| プロセス図スライド（学習パス） | ②価値提供 | 角丸ノードの形、矢印は下向き固定、ノード配色ルール（通常ステップ＝ターコイズ／分岐後の到達点＝オレンジ） | ステップ数、各ノードの文言、分岐先の名称（Roblox／Python等） |
| 比較スライド（✕／○型） | ②価値提供／③信頼構築 | 2カラム比較表の枠、ヘッダーチップ（英単語＋背景色）、✕／○の記号アイコン（色で判定させない） | 比較対象名、比較項目の文言、✕／○の判定 |
| リスト／ステップスライド | ②価値提供 | カード枠、番号バッジの形 | 箇条書き本文（準備リスト、学ぶ意味の解説など） |
| Before/Afterスライド | ③信頼構築 | 左右分割レイアウト、Before／Afterラベル位置 | 生徒作品の画像、コメント文 |
| 保護者の声カード | ③信頼構築 | 吹き出し／カード枠、アイコン位置 | 声の本文、属性（学年・匿名可） |
| 募集CTAスライド | ④行動喚起 | ボタン風デザイン、日時／人数（or期限）／申込み方法の3点表示枠（固定順序・省略不可） | 実際の日程、人数、申込みリンク・方法 |
| サンキュー／フォロー誘導 | 全カテゴリ末尾 | フォロー導線デザイン | メッセージ文言のみ |

---

## 2. カテゴリ別・使用パターンの目安

| カテゴリ | 比率 | 主に使うスライドパターン |
|---|---|---|
| ①認知（教室の独自性） | 2割 | 表紙／教室紹介カード |
| ②価値提供（専門性） | 4割 | プロセス図スライド、比較スライド、リスト／ステップスライド |
| ③信頼構築（実績） | 2割 | Before/Afterスライド、保護者の声カード、（比較スライドの流用も可） |
| ④行動喚起（導線） | 2割 | 募集CTAスライド、サンキュー／フォロー誘導 |

---

## 3. 運用ルール

1. 新しい投稿を作るときは、まず上の表から「どのスライド種別を使うか」をカテゴリ比率に沿って選ぶ
2. 固定要素はCanvaテンプレートをコピーして使い、可変要素だけ差し替える（ゼロからデザインしない）
3. 募集CTAスライドは日時・人数（or期限）・申込み方法の3点セットを必ず埋める。1つでも未確定な場合は、価値提供カテゴリのソフトCTA（プロフィール導線のみ）で代替し、日程確定後に本テンプレートへ差し替える
